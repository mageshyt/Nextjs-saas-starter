### Overview

This guide shows you how to set up user authentication using [Clerk](https://clerk.com/) in your **Next.js SaaS Starter**. Clerk provides out-of-the-box sign-up, sign-in, account management, and more.

### Prerequisites

- A [Clerk](https://clerk.com/) account
- Environment variables added to your `.env` file

Refer to the [Installation guide](https://clerk.com/docs/quickstarts/nextjs) if you haven't set up your project yet.

## 1. Clerk Dashboard Setup

1. Sign in to your Clerk dashboard at [clerk.com](https://clerk.com/).


2. Create a new application and note down your credentials:
   - Publishable Key (`NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY`)
   - Secret Key (`CLERK_SECRET_KEY`)

### 2. Environment Variables

Open your `.env` file (copied from `.env.example`) and add the following:

```bash title=".env"
NEXT_PUBLIC_APP_URL=http://localhost:3000
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_XXXXXXXXXXXX
CLERK_SECRET_KEY=sk_test_XXXXXXXXXXXX
```

<Callout type="warning" twClass="mt-4"> These keys are sensitive. Do not commit them to version control. Use environment variables on your hosting platform. </Callout>


### 3. Project Configuration

1. Install Clerk in your project:

```bash
bun add @clerk/clerk-react
```
*(Or use npm / pnpm / yarn depending on your setup.)*



### 4. Webhook Configuration

1. Go to your Clerk dashboard and navigate to **configuration** > **Webhooks**.

  src="https://oeb69r4qet.ufs.sh/f/hPqdRuZjNogwQLI59frthiWaqxAr6knfFlocLUgvVJKG9zm2"
  className="rounded-lg my-6"
  alt="Webhook configuration"
/>

2. Add a new webhook with the following details:
   - **URL**: `https://yourdomain.com/api/clerk/webhooks/user`

3. Turn on the **User Events** 

  - **User Created**
  - **User Updated**
  - **User Deleted**
